[[configure-snapmirror-active-sync-for-nas]]
== NAS용 SnapMirror active sync 구성

ONTAP 9.19.1 버전부터 NAS 구성에 대해 SnapMirror 활성 동기화를 설정할 수 있습니다.

.시작하기 전에
* SnapMirror 활성 동기화 관계에 있는 두 클러스터 모두 ONTAP 9.19.1 이상 버전을 실행해야 합니다.
* 클러스터 관리자여야 합니다.
* ONTAP Mediator 또는 ONTAP Cloud Mediator를 구성해야 합니다.
* 소스 및 타겟 클러스터를 내다봐야 합니다.


.NAS용 SnapMirror 활성 동기화에서 ONTAP 지원 및 미지원 기능
다음 표는 NAS용 SnapMirror 액티브 싱크에서 지원되는 ONTAP 기능과 지원되지 않는 ONTAP 기능을 설명합니다.

|===
| ONTAP 기능 | SnapMirror 액티브 싱크 지원 | 추가 정보 


| 애플리케이션 일관성 그룹 | 아니요 |  


| 감사 | 예 | SVM에서 통합된 로그만 복제됩니다. 


| DP 볼륨 | 아니요 |  


| FabricPool | 부분적으로 | The `all` 티어링 정책은 SnapMirror 액티브 싱크 NAS 볼륨에서 허용되지 않습니다. 


| 파일 시스템 분석 | 부분적으로 | 보조 SVM에서 파일 시스템 분석은 기본적으로  `Off`으로 설정되어 있으며, 장애 조치 후에는 수동으로  `On`으로 설정해야 합니다. 


| FlexCache 볼륨 | 아니요 |  


| FlexGroup 볼륨 | 아니요 |  


| Fpolicy 및 바이러스 백신 검사 | 예 |  


| 로드 공유 미러 | 아니요 | ONTAP 9.19.1에서 NAS용 SnapMirror 액티브 동기화는 AFF 클러스터에서 2개의 노드만 지원합니다. 


| MetroCluster 구성 | 아니요 |  


| NAS 명령에 대한 다중 관리자 검증 | 아니요 |  


| S3 NAS | 아니요 |  


| SnapMirror 액티브 싱크 NAS로 보호되는 SVM의 SAN 볼륨 | 아니요 | SnapMirror 활성 동기화의 NAS 초기화는 LUN을 호스팅하는 볼륨이 SVM에 존재하는 경우 실패합니다. 


| SnapLock | 아니요 |  


| 클라우드로의 SnapMirror | 아니요 |  


| SVM 마이그레이션 | 아니요 |  
|===
.이 작업에 대해
SnapMirror 활성 동기화를 NAS에 사용하려면 대상 SVM을 생성해야 합니다. 대상 SVM은 장애 조치가 발생할 때까지 비활성 상태로 유지됩니다.

.단계
NAS용 SnapMirror 활성 동기화를 구성하려면 System Manager 또는 ONTAP CLI를 사용할 수 있습니다.

[role="tabbed-block"]
====
.시스템 관리자
--
. 소스 클러스터에서 클러스터 > 스토리지 VM으로 이동합니다.
. 구성하려는 SVM 이름 옆의 image:icon_kabob.gif["메뉴 옵션 아이콘"]을 클릭하고 *보호*를 선택하십시오.
. *스토리지 VM 보호* 창에서 타겟 클러스터를 선택합니다.
.  `AutomatedFailOver`을(를) 복제 정책으로 선택합니다.
. 대상 위치에 생성될 스토리지 VM의 이름을 입력하십시오.
. 보호할 볼륨을 선택합니다:
+
.. *모든 볼륨 보호* – 기본적으로 모든 볼륨이 보호됩니다.
.. *보호할 볼륨 선택* - 선택한 볼륨만 보호됩니다
.. *제외할 볼륨을 선택하세요* - 선택한 볼륨은 보호 대상에서 제외됩니다


. *관계 초기화*를 클릭합니다.
. 저장 * 을 클릭합니다.
. *스토리지 VM 보호* 창에서 *보호* 열의 SVM 보호 상태를 확인하십시오.


--
.CLI를 참조하십시오
--
. 타겟 클러스터에서 dp-destination 유형의 SVM을 생성합니다.
+
`vserver create -vserver <SVM name> -sub-type dp-destination`

+
예:

+
[listing]
----
destination::> vserver create -vserver vs_b -sub-type dp-destination
----
. 소스 SVM에 대한 SVM 피어 관계를 생성합니다.
+
`vserver peer create -vserver <destination SVM> -peer-vserver <souce SVM> -peer-cluster <source cluster> -applications snapmirror`

+
예:

+
[listing]
----
destination::> vserver peer create -vserver vs_b -peer-vserver vs_a -peer-cluster SiteA -applications snapmirror
----
. 소스 SVM에서 피어 요청을 수락합니다.
+
`vserver peer accept -vserver <source SVM name> -peer-vserver <destination SVM name>vserver peer accept -vserver <source SVM name> -peer-vserver <destination SVM name>`

+
예:

+
[listing]
----
source::> vserver peer accept -vserver vs_a -peer-vserver vs_b
----
. SnapMirror 관계를 생성합니다.
+
`snapmirror create -source-path <source SVM name>: -destination-path <destination SVM name>: -policy AutomatedFailOver`

+
예:

+
[listing]
----
destination::> snapmirror create -source-path vs_a: -destination-path vs_b: -policy AutomatedFailOver
----
. 관계가 생성되었는지 확인하고 상태를 점검하십시오.
+
'snapmirror show'

+
예:

+
[listing]
----
destination::> snapmirror show
Progress
	Source            Destination Mirror        Relationship   Total             Last
	Path        Type  Path        State         Status         Progress  Healthy Updated
	----------- ---- ------------ -------       -------------- --------- ------- --------
	vs_a:       XDP  vs_b:        Uninitialized Idle           -         true    -

----
. 대상 SVM에서 관계를 초기화합니다.
+
`snapmirror initialize -destination-path <SVM name>:`

+
예:

+
[listing]
----
destination::> snapmirror initialize -destination-path vs_b:
----
. 초기화 상태를 확인합니다.
+
스냅미러 쇼

+
예:

+
[listing]
----
	destination::> snapmirror show
	                                                                             Progress
	Source            Destination Mirror        Relationship   Total             Last
	Path        Type  Path        State         Status         Progress  Healthy Updated
	----------- ---- ------------ -------       -------------- --------- ------- --------
	vs_a:       XDP  vs_b:        Snapmirrored  InSync         -         true    -

----


--
====


[[create-an-unprotected-volume-in-a-snapmirror-active-sync-nas-configuration]]
== SnapMirror 액티브 싱크 NAS 구성에서 보호되지 않은 볼륨 생성

보호가 필요하지 않은 경우 SnapMirror 활성 동기화에서 볼륨을 제외할 수 있습니다. 예를 들어 개발 테스트를 위해 FlexClone 볼륨을 생성할 때 사용할 수 있습니다.

.단계
시스템 관리자 또는 ONTAP CLI를 사용하여 복제된 볼륨을 NAS용 SnapMirror 활성 동기화에서 제외할 수 있습니다.

[role="tabbed-block"]
====
.시스템 관리자
--
. 소스 클러스터에서 *스토리지 > 볼륨*으로 이동합니다.
. 볼륨 목록에서 복제하려는 볼륨 이름 옆의 image:icon_kabob.gif["메뉴 옵션 아이콘"]을 클릭하고 *복제*를 선택합니다.
. *볼륨 복제* 창에서 *비즈니스 연속성을 위한 보호에 복제본 포함* 옵션이 선택 해제되어 있는지 확인하십시오.
. *복제*를 클릭합니다.


--
.CLI를 참조하십시오
--
. 소스 클러스터에서 기존 SnapMirror 활성 동기화 구성에 보호되지 않은 볼륨을 추가합니다.
+
`volume clone create -vserver <SVM name> -flexclone <clone vol name> -parent-vserver <SVM name> -parent-volume <vol name> -parent-snapshot <snapshot name>`
`volume add -vserver <SVM name> -volume <volume name> -smas-protection unprotected`



--
====


[[use-the-ontap-cli-to-exclude-a-volume-from-a-snapmirror-active-sync-nas-configuration]]
== ONTAP CLI를 사용하여 SnapMirror 활성 동기화 NAS 구성에서 볼륨 제외

더 이상 보호가 필요하지 않거나 볼륨을 삭제하려는 경우 SnapMirror 액티브 싱크 NAS 구성에서 볼륨을 제외할 수 있습니다.


NOTE: 현재 보호된 볼륨을 보호 해제로 표시하면 SnapMirror 활성 동기화 관계에 대한 축소 작업이 시작됩니다. 이 축소 작업은 SnapMirror 소스 및 타겟 클러스터 모두에서 해당 볼륨에 대한 SnapMirror 항목을 삭제합니다.


NOTE: 볼륨을 보호 해제로 표시해도 기본 또는 보조 관계에서 해당 볼륨이 자동으로 삭제되지는 않습니다. 볼륨을 삭제하려면 기본 및 보조 SVM 모두에서 수동으로 삭제해야 합니다.

.단계
. SnapMirror 활성 NAS 보호에서 볼륨을 제거합니다.
+
`volume modify -vserver <SVM name> -volume <volume name> -smas-protection unprotected`





[[convert-an-svm-dr-relationship-to-a-snapmirror-active-sync-relationship]]
== SVM DR 관계를 SnapMirror 활성 동기화 관계로 변환합니다

ONTAP 9.19.1에서는 SVM DR 관계를 NAS용 SnapMirror 활성 동기화 관계로 직접 변환할 수 없습니다. 대신 SVM DR 관계를 삭제하고 새 대상 SVM을 생성한 다음 새 SnapMirror 활성 동기화 관계를 설정하고 관계를 초기화해야 합니다.


NOTE: 해당  `snapmirror resync` 명령어는 현재 지원되지 않습니다.

. 타겟 클러스터에서 SnapMirror 관계를 삭제합니다.
+
`snapmirror delete -destination-path`

. 기본 클러스터에서 관계 정보를 해제합니다.
+
`snapmirror release -relationship-info-only true`

. 타겟 클러스터에서 dp-destination 하위 유형을 사용하여 SVM을 생성합니다.
+
`vserver create -vserver <SVM name> -sub-type dp-destination`

. 타겟 클러스터에서  `AutomatedFailover` 정책을 사용하여 SnapMirror 관계를 생성합니다.
+
`snapmirror create -source-path <source SVM name> -destination-path <destination SVM name> -policy AutomatedFailOver`

. 타겟 클러스터에서 관계를 초기화합니다.
+
`snapmirror initialize -destination-path <SVM name>`
