= ONTAP 일관성 그룹 또는 NAS SVM 간에 공통 스냅샷 생성
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
정기적으로 예약된 스냅샷 작업 외에도 기본 SnapMirror 일관성 그룹의 볼륨과 보조 SnapMirror 일관성 그룹의 볼륨 사이에 SAN 구성의 경우 또는 NAS 구성의 경우 SVM 수준에서 공통 link:../concepts/snapshot-copies-concept.html["스냅샷"]을 수동으로 생성할 수 있습니다.

.이 작업에 대해
자동 페일오버 정책의 예약된 스냅샷 생성 간격은 12시간이며, 자동 페일오버 듀플렉스 정책의 예약된 스냅샷 생성 간격은 3시간입니다.

.시작하기 전에
* SnapMirror 그룹 관계가 동기화되어 있어야 합니다.


.단계
시스템 관리자 또는 ONTAP CLI를 사용하여 공통 스냅샷을 생성할 수 있습니다.

[role="tabbed-block"]
====
.시스템 관리자
--
. 타겟 클러스터에서 *보호 > 복제*로 이동합니다.
. 관계 소스를 찾아 image:icon_kabob.gif["메뉴 옵션 아이콘"]을 선택하고 *업데이트*를 선택합니다.
. *정책대로* 또는 *스냅샷 선택*을 선택하여 기존 스냅샷을 선택합니다.
. *업데이트*를 선택합니다.


--
.CLI를 참조하십시오
--
. 대상 사이트에서 공통 스냅샷을 생성합니다.
+
`snapmirror update -destination-path <destination path>`

+
SAN 구성의 예:

+
[listing]
----
destination::>snapmirror update -destination-path vs1_dst:/cg/cg_dst
----
+
NAS 구성의 예:

+
[listing]
----
snapmirror update -destination-path vs_b:
----
. 업데이트 진행 상황 모니터링:
+
`destination::>snapmirror show -fields newest-snapshot`
