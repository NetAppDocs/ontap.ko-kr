= PRR 및 RACK을 사용하여 ONTAP TCP WAN 성능 향상
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
ONTAP 9.19.1 버전부터 ONTAP은 PRR과 RACK이라는 두 가지 손실 복구 기능을 ONTAP TCP 스택에 통합하여 손실이 많고 지연 시간이 긴 WAN 링크에서 TCP 데이터 전송 성능을 향상시킵니다. PRR은 모든 네트워크에 적합하며 기본적으로 활성화되어 있습니다. RACK은 링크 속도가 약 25Gigabits/초인 혼잡한 네트워크에 적합하며 기본적으로 비활성화되어 있습니다.

* *비례 전송률 감소(PRR):* 손실 복구 중 감소된 혼잡 윈도우에 최대한 가깝게 복구 데이터를 전송하여, 활용률 저하 및 과도한 재전송 버스트를 방지합니다.
* *최근 확인 응답(RACK)*: 기존의 중복 ACK 카운팅 방식 대신 시간 기반 감지 방식을 사용하여 패킷이 순서대로 도착하지 않더라도 손실된 패킷을 더 빠르고 정확하게 식별합니다. RACK에는 테일 손실 프로브(TLP) 기능이 포함되어 있어 특정 RPC의 마지막 패킷 중 하나가 손실되어 이전 요청이 완료된 후에야 새로운 데이터를 사용할 수 있는 경우에도 트랜잭션 통신을 더 빠르게 완료할 수 있습니다. 이를 통해 WLAN(WiFi), WAN, 인터넷 VPN 및 클라우드 터널과 같은 열악한 네트워크 환경에서 요청 완료 속도를 높이고 불필요한 재전송을 줄일 수 있습니다.


PRR 및 RACK은 사용 중인 프로토콜에 구애받지 않고 NFS, SMB, iSCSI, NVMe/TCP 및 S3 액세스를 개선할 뿐만 아니라 WAN 연결을 통한 SnapMirror 및 FlexCache를 포함하여 클러스터 간 네트워크를 사용하는 ONTAP 기능의 성능을 향상시킵니다. 이러한 기능은 모든 NetApp 스토리지 시스템에서 지원됩니다.


NOTE: RACK 활성화는 링크 속도가 약 25기가비트/초인 혼잡한 네트워크 환경에서만 권장됩니다. 링크 속도가 25기가비트/초 이상인 원활한 네트워크 환경에서는 RACK을 사용해도 표준 스택과 동일한 처리량을 얻지 못할 수 있습니다.

.단계
PRR은 기본적으로 활성화되어 있습니다. RACK은 기본적으로 비활성화되어 있습니다. 사용 환경이 RACK에 적합한 경우 다음 단계를 통해 활성화할 수 있습니다.

. IPspace에 대해 RACK TCP 스택 기능을 활성화합니다.
+
[source, cli]
----
network ipspace modify -ipspace <ipspace_name> -tcp-stack rack
----
+

NOTE: 이 변경 사항은 새로운 연결에만 적용됩니다. IPspace에 대한 기존 연결에서 RACK을 사용하려면 노드 간에 LIF를 마이그레이션하거나 LIF를 일시적으로 비활성화한 후 다시 활성화해야 합니다.

. IPspace에 변경 사항이 적용되었는지 확인하십시오.
+
[source, cli]
----
network ipspace show -fields tcp-stack
----
