= ONTAP 9.19.1에서 되돌리기 전에 ONTAP 9.19.1 이상에서 생성된 QoS 정책 그룹을 수정합니다
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
적응형 및 비적응형 QoS 정책 그룹은 기본적으로 워크로드의 처리량 상한을 1초 동안 50%까지 일시적으로 높여 단기적인 성능 향상 요구에 대응하도록 자동으로 활성화됩니다. ONTAP 9.19.1부터는 이러한 기본 설정을 수정할 수 있습니다. ONTAP 9.18.1 이하 버전에서는 이러한 설정을 변경할 수 없습니다. ONTAP 9.19.1 이상 버전에서 생성한 QoS 정책 그룹의 기본 설정을 수정한 경우, ONTAP 9.19.1에서 ONTAP 9.18.1로 되돌리기 전에 설정을 원래대로 되돌려야 합니다.

.단계
. 자동 처리량 상한 증가를 위해 활성화된 QoS 정책 그룹이 있는지 확인하십시오.
+
[source, cli]
----
qos policy-group show – instance
----
.  `burst-percent`,  `burst-iops` 및  `burst-duration`로 구성된 정책 그룹이 있는 경우 이러한 옵션을 0으로 설정하십시오.
+
[source, cli]
----
qos policy-group modify -policy-group <policy-group> -burst-percent 0 -burst-iops 0 -burst-duration 0
----
