= SnapMirror 클라우드 백업을 객체 스토리지에 저장
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
_SnapMirror Cloud_는 데이터 보호 워크플로우를 클라우드 또는 온프레미스 객체 스토리지 솔루션으로 전환하려는 ONTAP 사용자를 위해 설계된 백업 및 복구 기술입니다. 기존의 테이프 백업 아키텍처에서 벗어나려는 조직은 객체 스토리지를 장기 데이터 보존 및 아카이빙을 위한 대체 저장소로 활용할 수 있습니다. SnapMirror Cloud는 증분 방식의 영구 백업 전략의 일환으로 ONTAP에서 객체 스토리지로의 복제를 제공합니다.

SnapMirror 클라우드 복제는 라이센스가 부여된 ONTAP 기능입니다. SnapMirror 클라우드는 SnapMirror 복제 기술 제품군의 확장 기능으로 ONTAP 9.8에서 도입되었습니다. SnapMirror는 ONTAP 간 백업에 자주 사용되지만, SnapMirror 클라우드는 동일한 복제 엔진을 사용하여 ONTAP에서 S3 호환 객체 스토리지 백업으로 스냅샷을 전송합니다.

백업 사용 사례를 위해 설계된 SnapMirror Cloud는 장기 보존 및 아카이브 워크플로우를 모두 지원합니다. SnapMirror와 마찬가지로, 초기 SnapMirror Cloud 백업은 볼륨의 기본 전송을 수행합니다. 이후 백업에서는 SnapMirror Cloud가 소스 볼륨의 스냅샷을 생성하고 변경된 데이터 블록만 포함하는 스냅샷을 오브젝트 스토리지 타겟으로 전송합니다.

SnapMirror Cloud는 ONTAP 시스템과 Amazon S3, Google Cloud Storage, Microsoft Azure Blob Storage를 포함한 일부 온프레미스 및 퍼블릭 클라우드 객체 스토리지 대상 간에 연결을 구성할 수 있습니다. 추가적인 온프레미스 객체 스토리지 대상으로는 StorageGRID 및 ONTAP S3가 있습니다.

ONTAP System Manager를 사용하여 SnapMirror Cloud 구성을 관리하는 것 외에도 SnapMirror Cloud 백업 관리를 위한 여러 오케스트레이션 옵션을 사용할 수 있습니다.

* SnapMirror 클라우드 복제를 지원하는 여러 타사 백업 파트너가 있습니다. 참여 업체 목록은 link:https://www.netapp.com/blog/new-backup-architecture-snapdiff-v3/["NetApp 블로그"^]에서 확인할 수 있습니다.
* ONTAP 환경을 위한 NetApp 네이티브 솔루션을 위한 NetApp 백업 및 복구
* 데이터 보호 워크플로우를 위한 맞춤형 소프트웨어를 개발하거나 자동화 툴을 활용하기 위한 API



NOTE: System Manager에서 관리하는 SnapMirror Cloud 복제는 오브젝트 스토리지 타겟으로 StorageGrid 및 ONTAP S3만 지원합니다. 다른 제공업체는 지원되지 않습니다.


NOTE: SnapMirror 클라우드 API 사용은 일반적으로 백업 파트너로 제한됩니다. 자체 API 기반 솔루션을 개발하려는 고객은 라이선스 조건에 대한 정보를 얻기 위해 NetApp 계정 담당 팀에 문의해야 합니다.
