= NVMe 복사 오프로드에 대해 알아보세요
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
NVMe 복사 오프로드를 사용하면 NVMe 호스트가 CPU에서 ONTAP 스토리지 컨트롤러의 CPU로 복사 작업을 오프로드할 수 있습니다. 호스트는 애플리케이션 워크로드를 위해 CPU 리소스를 예약하는 동시에 한 NVMe 네임스페이스에서 다른 네임스페이스로 데이터를 복사할 수 있습니다.

예를 들어, 성능 분포를 개선하기 위해 스토리지 작업 부하를 재조정해야 한다고 가정해 보겠습니다.  이를 위해서는 각각 평균 크기가 500GB인 45개의 NVMe 네임스페이스를 포함하는 10개의 가상 머신(VM)을 마이그레이션해야 합니다.  즉, 약 22.5TB의 데이터를 복사해야 한다는 의미입니다.  호스트는 데이터 마이그레이션에 자체 CPU를 사용하는 대신 NVMe 복사 오프로드를 사용하여 데이터가 복사되는 동안 애플리케이션 워크로드에 대한 CPU 리소스가 감소하지 않도록 할 수 있습니다.



== NVMe 복사 오프로드 지원 및 제한 사항

NVMe 복사 오프로드는 ONTAP 9.18.1부터 지원됩니다.  ONTAP NVMe 복사 오프로드를 시작할 수 없습니다. 호스트에서 지원하고 시작해야 합니다.

ONTAP 사용한 NVMe 복사 오프로드 작업에는 다음과 같은 제한 사항이 적용됩니다.

* 지원되는 최대 복사 작업 크기는 16MB입니다.
* 데이터는 동일한 하위 시스템 내의 NVMe 네임스페이스 간에만 마이그레이션될 수 있습니다.
* 데이터는 동일한 HA 쌍의 노드 간에만 마이그레이션될 수 있습니다.
