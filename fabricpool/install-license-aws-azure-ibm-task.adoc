= ONTAP 클러스터에 FabricPool 라이센스를 설치합니다
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
NetApp Cloud Tiering은 2026년 4월 26일부로 구매 또는 라이선스 갱신이 더 이상 불가능합니다. NetApp Cloud Tiering을 사용 중인 경우, NetApp은 NetApp 담당자와 협력하여 기존 티어링 라이선스를 ONTAP FabricPool 라이선스로 전환하는 것을 권장합니다.

기존 고객은 구독 또는 라이선스 계약이 만료될 때까지 NetApp Cloud Tiering을 계속 사용하고 지원을 받을 수 있습니다. 구독이 만료된 후에는 NetApp Cloud Tiering 기능이나 지원에 더 이상 액세스할 수 없습니다.

현재 Cloud Tiering 라이선스를 사용 중인 경우 Cloud Tiering 라이선스가 만료되거나 최대 용량에 도달할 때까지는 영향을 받지 않습니다. 라이선스를 업데이트해야 할 때 또는 그 전에 NetApp에 문의하여 클라우드로 데이터를 티어링하는 기능에 중단이 발생하지 않도록 하십시오.

link:https://docs.netapp.com/us-en/data-services-cloud-tiering/task-licensing-cloud-tiering.html["NetApp Cloud Tiering 라이선스 변경 사항에 대해 자세히 알아보기"^].

System Manager를 사용하여 FabricPool 라이센스를 다운로드하고 활성화할 수 있습니다.

FabricPool 라이센스는 클러스터 차원의 라이센스입니다. 클러스터에서 FabricPool와 연결된 오브젝트 스토리지에 대해 구입한 사용 제한이 포함됩니다. 클러스터 전체의 사용량이 사용 제한 용량을 초과해서는 안 됩니다. 라이센스의 사용 한도를 늘려야 하는 경우 영업 담당자에게 문의하십시오.

FabricPool 라이센스는 영구 또는 기간 기반, 1년 또는 3년 형식으로 제공됩니다.

기존 클러스터 구성에 대한 최초 FabricPool 주문 시 10TB의 무료 용량이 제공되는 기간제 FabricPool 라이선스가 제공됩니다. 영구 라이선스에는 무료 용량이 제공되지 않습니다. 클라우드 계층에 NetApp StorageGRID 또는 ONTAP S3를 사용하는 경우 라이선스가 필요하지 않습니다. Cloud Volumes ONTAP는 사용 중인 공급업체와 관계없이 FabricPool 라이선스가 필요하지 않습니다.

.시작하기 전에
NetApp 지원 사이트에서 NetApp 라이센스 파일(NLF)을 받으려면 클러스터 UUID가 필요합니다.

.단계
이 작업은 System Manager를 사용하여 클러스터에 라이센스 파일을 업로드하는 경우에만 지원됩니다.

. FabricPool 라이센스용 NetApp 라이센스 파일(NLF)을 에서 다운로드하십시오 link:https://mysupport.netapp.com/site/global/dashboard["NetApp Support 사이트"^].
. System Manager를 사용하여 FabricPool 라이센스를 클러스터에 업로드하려면 다음 작업을 수행합니다.
+
.. 클러스터 > 설정 * 창의 * Licenses * 카드에서 을 클릭합니다 image:icon_arrow.gif["화살표 아이콘"].
.. 라이센스 * 페이지에서 를 image:icon_add.gif["추가 아이콘"]클릭합니다.
.. 라이센스 추가 * 대화 상자에서 * 찾아보기 * 를 클릭하여 다운로드한 NLF를 선택한 다음 * 추가 * 를 클릭하여 파일을 클러스터에 업로드합니다.
