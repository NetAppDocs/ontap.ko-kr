= ONTAP 시스템 관리자를 사용하여 NFS를 사용하여 Linux 서버에 NAS 스토리지 활성화
:hardbreaks:
:toclevels: 1
:allow-uri-read: 
:toclevels: 1
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
스토리지 VM을 생성하거나 수정하여 NFS 서버에서 Linux 클라이언트에 데이터를 제공할 수 있도록 합니다.

다음 절차를 사용하여 NFS 프로토콜에 새 스토리지 VM 또는 기존 스토리지 VM을 사용하도록 설정합니다.

image:workflow_nas_enable_linux_nfs.png["워크플로 요약: 1 NFS 2 활성화 루트 볼륨 내보내기 정책 열기 3 DNS 4 구성 이름 서비스 구성 5 필요한 경우 암호화 구성"]

.시작하기 전에
사용자 환경에 필요한 네트워킹, 인증 또는 보안 서비스에 대한 구성 세부 정보를 기록해 두십시오.

.단계
. 스토리지 VM에서 NFS를 사용하도록 설정합니다.
+
** 새 스토리지 VM의 경우: *스토리지 > 스토리지 VM*을 선택하고 *추가*를 선택한 다음 스토리지 VM 이름을 입력하고 *SMB/CIFS, NFS, S3* 탭에서 *NFS 활성화*를 선택합니다.
+
... 기본 언어를 확인합니다.
... 네트워크 인터페이스를 추가합니다.
... 스토리지 VM 관리자 계정 정보 업데이트(선택 사항)


** 기존 스토리지 VM의 경우: *클러스터 > 스토리지 VM*을 선택하고 스토리지 VM을 선택한 다음 *설정*을 선택하고 *NFS* 아래에서 image:icon_gear.gif["작업 아이콘"] 또는 image:icon-edit-pencil-blue-outline.png["편집 아이콘"]를 선택한 후 *NFS 활성화*를 선택합니다.
+
... 필요에 따라 특정 NFS 프로토콜 버전 및 옵션을 활성화 또는 비활성화합니다.
+

NOTE: NFSv4.2 파일 액세스 이벤트에 대한 FPolicy 모니터링은 지원되지 않습니다. FPolicy 모니터링을 구성하려면 SVM에서 NFSv4.2를 비활성화해야 합니다. 자세한 내용은 link:./nfs-admin/enable-disable-nfs-task.html["ONTAP SVM에 대해 NFS를 활성화 또는 비활성화합니다"]를 참조하십시오.





. 스토리지 VM 루트 볼륨의 엑스포트 정책을 엽니다.
+
.. *Storage > Volumes*를 선택하고 스토리지 VM의 루트 볼륨(기본값은 _volume-name_ _root)을 선택한 다음 *Export Policy* 아래에 표시되는 정책을 선택합니다.
.. 규칙을 추가하려면 *추가*를 선택하세요.
+
*** 클라이언트 사양 = 0.0.0.0/0
*** 액세스 프로토콜 = NFS
*** 액세스 세부 정보 = UNIX 읽기 전용




. 호스트 이름 확인을 위한 DNS 구성: *스토리지 > 스토리지 VM*을 선택하고 스토리지 VM을 선택한 다음 *설정*을 선택하고 *DNS*에서 image:icon_gear.gif["작업 아이콘"]을(를) 선택합니다.
. 필요에 따라 이름 서비스를 구성합니다.
+
.. *Storage > Storage VMs*를 선택하고 스토리지 VM을 선택한 다음 *Settings*를 선택하고 image:icon_gear.gif["작업 아이콘"] LDAP 또는 NIS를 선택합니다.
.. 이름 서비스 스위치 타일에서 image:icon_pencil.gif["편집 아이콘"]을(를) 선택하여 변경 사항을 포함하십시오.


. 필요한 경우 Kerberos 구성:
+
.. *Storage > Storage VMs*를 선택하고 스토리지 VM을 선택한 다음 *Settings*를 선택합니다.
.. Kerberos 타일에서 image:icon_arrow.gif["화살표 아이콘"]를 선택한 다음 *추가*를 선택합니다.
