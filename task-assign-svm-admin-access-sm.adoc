= ONTAP System Manager를 사용하여 SVM 관리자 액세스 권한 할당
:hardbreaks:
:toclevels: 1
:allow-uri-read: 
:toclevels: 1
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
ONTAP 9.19.1부터 ONTAP 클러스터 관리자는 SVM 관리자라고 하는 SVM을 관리하는 사람에게 System Manager 액세스 권한을 할당할 수 있습니다.

이는 특히 테넌트 기반 ONTAP 환경에서 유용하며, 클러스터 관리자가 각 테넌트에 대한 SVM dashboard를 제공할 수 있습니다.

클러스터 관리자는 SVM 관리자에게 볼륨 생성 권한을 부여할 수도 있습니다.



[[step-1-prepare-the-management-network-interface-for-the-svm-dashboard]]
== 1단계: SVM 대시보드에 대한 관리 네트워크 인터페이스 준비

SVM 관리자에게 SVM 대시보드를 할당하기 전에 SVM에 관리 네트워크 인터페이스가 있는지 확인하십시오. SVM에 네트워크 인터페이스가 없는 경우 네트워크 인터페이스를 생성할 수 있습니다.

.단계
. System Manager에서 *네트워크 인터페이스 > 개요*를 선택한 다음 *추가*를 선택하여 네트워크 인터페이스를 생성합니다.
. 네트워크 인터페이스 유형으로 *Storage VM management*를 선택하십시오.
. SVM 관리 네트워크 인터페이스의 *IP 주소* 및 *서브넷 마스크*를 지정합니다.




[[step-2-verify-the-vsadmin-user-account-configuration]]
== 2단계: vsadmin 사용자 계정 구성 확인

SVM이 생성될 때마다 ONTAP는 자동으로 연결된 vsadmin 사용자 계정을 생성합니다. 관리자에게 SVM 대시보드를 할당하기 전에 vsadmin 사용자 계정에 암호가 설정되어 있고 잠금 해제되어 있는지 확인하십시오.

.단계
. 클러스터 관리자로 System Manager 대시보드를 엽니다.
. *Cluster > Storage VMs*를 선택합니다.
. SVM을 선택하고 *Settings*를 선택한 다음 *Security > Users and roles*를 선택합니다.
. *vsadmin* 사용자 옆에서 image:icon_kabob.gif["메뉴 옵션 아이콘"]를 선택하고, 필요한 경우 다음을 선택합니다.
+
.. vsadmin 암호를 업데이트하려면 *Change password*를 클릭하세요.
.. 사용 가능한지 확인하려면 *잠금 해제*하십시오.






[[step-3-allow-svm-administrators-to-add-volumes-optional]]
== 3단계: SVM 관리자가 볼륨을 추가할 수 있도록 허용합니다(선택 사항)

ONTAP 클러스터 관리자는 SVM 관리자가 자신의 SVM에 로컬 볼륨을 추가할 수 있도록 하려면 권한을 부여할 수 있습니다.

.단계
. System Manager 대시보드를 열고 클러스터 관리자로 Sign in합니다.
. *클러스터*에서 *스토리지 VM*을 선택합니다.
. 해당 SVM에 대해 image:icon_kabob.gif["메뉴 옵션 아이콘"]을(를) 선택합니다.
. 편집 * 을 선택합니다.
. *Resource Allocation*에서 *Limit volume creation to preferred local tiers* 확인란을 선택합니다.
. 기본적으로 확인란을 선택하면 모든 로컬 계층이 포함됩니다. 필요한 경우 특정 계층을 제거할 수 있습니다. 모든 로컬 계층을 제거하거나 확인란을 비워 두면 SVM 관리자는 로컬 볼륨을 프로비저닝할 수 없습니다.
. 저장 * 을 선택합니다.




[[step-4-assign-svm-administrator-access]]
== 4단계: SVM 관리자 액세스 권한 부여

SVM 대시보드 액세스 권한을 할당하려면 각 SVM 관리자에게 SVM 관리 IP 주소와 vsadmin 암호를 제공하십시오.

.단계
. System Manager에서 *Network interfaces > Overview*를 선택한 다음 *Management* 네트워크 인터페이스를 선택합니다.
. IP 주소를 선택하고 *vsadmin* 사용자 이름 및 암호와 함께 SVM 관리자와 공유합니다.
